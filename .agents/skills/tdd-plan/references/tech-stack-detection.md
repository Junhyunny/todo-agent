# 기술 스택 파일 연동 가이드

`tdd-plan`은 이 문서의 규칙으로 직접 현재 프로젝트의 기술 스택을 추론하고, 필요하면 `.agents/tech-stack.md`를 직접 생성/갱신한다.

---

## 실행 순서

1. 기존 `.agents/tech-stack.md`가 있으면 먼저 읽는다
2. 파일이 없거나 오래되었다고 판단되면 현재 프로젝트를 직접 스캔해 갱신한다
3. `## 스택 요약`에서 현재 프로젝트의 영역과 스택을 파악한다
4. 스토리와 가장 관련된 영역의 상세 섹션을 읽어 프레임워크와 작업 경로를 정한다

---

## 읽어야 할 값

`tdd-plan`은 `.agents/tech-stack.md`에서 다음 값만 추출하면 된다:

- **Stack**
- **Language**
- **Build tool**
- **Unit test framework**
- **E2E framework**
- **Path**
- **Source roots**
- **Test roots**

긴 감지 로그를 다시 만들지 않는다.

---

## 모노레포 처리

스택 요약 표에 여러 영역이 있으면:

1. 스토리와 같은 경로/도메인과 가장 가까운 영역을 우선 선택한다
2. 애매하면 개발자에게 질문한다
3. 선택한 영역의 상세 섹션만 후속 단계에서 참조한다

---

## 재동기화가 필요한 경우

다음 상황이면 현재 프로젝트를 다시 스캔해 `.agents/tech-stack.md`를 갱신한다:

- 빌드 파일이 새로 생김
- 새 프레임워크가 추가됨
- 작업 대상 서비스가 바뀜
- 기존 `.agents/tech-stack.md`가 없거나 오래되었음

세부 감지 규칙과 파일 형식은 이 디렉토리의 문서를 따른다.
