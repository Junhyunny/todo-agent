# `ARCHITECTURE.md` 파일 형식

`/sync-architecture`가 생성/갱신하는 프로젝트 아키텍처 파일의 형식을 정의한다.

---

## 파일 위치

```text
ARCHITECTURE-{AREA}.md   # 다중 영역 (예: ARCHITECTURE-FRONTEND.md, ARCHITECTURE-LAMBDA.md)
ARCHITECTURE.md          # 단일 영역 (루트에 빌드 파일이 바로 있는 경우)
```

프로젝트 루트에 위치하며, AI 에이전트가 탐색 없이 해당 영역의 구조를 파악할 수 있는 **영역별 지형도**다.

---

## 작성 원칙

- **구조·흐름·경계**만 담는다. 배경 설명·히스토리·TBD는 포함하지 않는다
- 에이전트가 파일을 열었을 때 바로 읽고 작업을 시작할 수 있는 밀도를 유지한다
- 수정 금지 파일/폴더는 `[수정 금지]`로 명시한다
- 없는 항목은 줄 자체를 생략한다

## 작성 규칙

1. 각 영역은 **디렉터리 구조 → 데이터 흐름 → 레이어 아키텍처 → 경계** 순서로 작성한다
2. 디렉터리 구조는 2~3 depth로 제한하고, 각 폴더/파일에 역할을 한 줄로 단다
3. 데이터 흐름은 화살표(`→`)로 단방향 경로를 표현한다
4. 경계 표는 수정 금지 항목이 있을 때만 포함한다
5. `## Notes` 섹션은 갱신 시 유지한다
6. 없는 섹션은 생략한다 — 내용 없이 제목만 남기지 않는다

---

## 갱신 규칙

1. 구조가 달라지면 해당 영역 섹션을 갱신한다
2. 사라진 영역은 파일에서도 제거한다
3. 새로 생긴 영역은 새 섹션으로 추가한다
4. `## Notes` 섹션은 유지한다
